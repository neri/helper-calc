# ライブコーディング用設計書: 助っ人貸出回数計算機

## 概要
このプロジェクトは、ユーザーが入力した「報酬」と「時間」から、「助っ人の貸し出し」回数を推定する計算機 (Single Page Application) を実装します。アプリはオフラインで動作し、外部通信を行いません。ライブコーディングのデモンストレーションとして、シンプルで理解しやすい設計を採用します。

## 技術スタック
- **言語**: TypeScript
- **ビルドツール**: Vite
- **パッケージマネージャー**: pnpm
- **フレームワーク**: なし (Vanilla TypeScript + HTML/CSS)
- **実行環境**: ブラウザ (SPA)

## 機能仕様
### 主要機能
- ユーザーが「現在の報酬」と「経過時間 (時間単位)」を入力。
- 入力に基づき、「貸し出し」回数を推定して表示。
- 推定結果はリアルタイムで更新 (入力変更時に再計算)。

### 計算ロジック
- **報酬増加のルール**:
  - 時間経過: 1分ごとに20ポイント増加。
  - 貸し出し: 1回あたり50,000ポイント増加。
- **時間の扱い**:
  - 入力は1時間単位 (例: 1時間、2時間)。
  - 実際の時間は1:00〜1:59の範囲 (60分〜119分) を考慮。
- **推定式**:
  - まず、推定貸出回数の下限を算出: 下限 = floor( (報酬 - (時間 × 60分) × 20) / 50,000 )
  - 次に、下限と入力時間から報酬の範囲を計算: 最小報酬 = (時間 × 60分) × 20 + 下限 × 50,000, 最大報酬 = ((時間 × 60) + 59) × 20 + 下限 × 50,000
  - 入力報酬が範囲内なら、推定貸出回数は下限で確定。
  - 範囲外なら、従来の min/max を計算: 総報酬 = (経過時間 × 60分) × 20 + (貸し出し回数 × 50,000), 時間範囲を考慮し、最小/最大回数を推定。

### UI/UX 仕様
- **入力フォーム**:
  - 報酬: 数値入力フィールド (必須、正の整数)。
  - 時間: 数値入力フィールド (必須、正の整数、単位: 時間)。
- **出力表示**:
  - 推定貸し出し回数: 最小値と最大値が同じ場合は「推定貸し出し回数: X回」、異なる場合は「推定貸し出し回数: X〜Y回」で表示。
  - エラーメッセージ: 無効な入力時 (例: 負の数、ゼロなど)。
- **レイアウト**:
  - シンプルな中央配置フォーム。
  - レスポンシブデザイン (モバイル対応)。
- **スタイル**: 基本的なCSSで装飾 (ボタン、フォームのスタイリング)。

## 実装計画
### プロジェクト構造
```
/
├── index.html          # メインHTMLファイル
├── src/
│   ├── main.ts         # エントリーポイント (TypeScript)
│   ├── app.ts          # アプリのメインクラス
│   ├── calculator.ts   # 計算ロジック
│   └── styles.css      # CSSファイル
├── package.json        # 依存関係
├── tsconfig.json       # TypeScript設定
└── vite.config.ts      # Vite設定
```

### 実装ステップ (ライブコーディング順)
1. **プロジェクト初期化**: pnpm init, Viteセットアップ。
2. **HTML構造**: index.html の作成 (フォームと出力エリア)。
3. **TypeScript設定**: tsconfig.json の確認。
4. **計算ロジックの実装**: calculator.ts で推定関数を作成。
5. **メインアプリの実装**: app.ts でDOM操作とイベントハンドリング。
6. **エントリーポイント**: main.ts でアプリ起動。
7. **スタイリング**: styles.css でUI装飾。
8. **テスト実行**: Vite dev server で動作確認。

### 考慮点
- **エラーハンドリング**: 入力値のバリデーション (数値チェック、範囲チェック)。
- **パフォーマンス**: 計算は軽量なので最適化不要。
- **拡張性**: 将来的に追加機能 (例: 詳細なログ) を考慮したモジュール化。
- **アクセシビリティ**: ラベル付け、キーボード操作対応。

## テストケース
- 入力: 報酬=100,000, 時間=1 → 推定: 約1〜2回 (計算: 最小60分: (100,000 - 60*20)/50,000 ≈1.76 → 2回? 待って、正確に調整)。
- 入力: 報酬=200,000, 時間=2 → 推定: 約2〜3回。
- エラー: 報酬=0, 時間=0 → 無効入力メッセージ。
